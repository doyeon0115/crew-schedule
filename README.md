# 📅 우리끼리 스케줄

친구들 근무/휴무 스케줄을 한눈에 보고, **다 같이 만날 수 있는 시간**을 자동으로 찾아주는 웹앱.
백엔드 서버 없이 Firebase 무료 티어로 실시간 공유돼요.

> 앱의 "✏️ 스케줄 수정" 탭에서 실제 시간으로 고치면 됩니다.

---

## 1단계 — 바로 써보기 (설정 0초)

`index.html` 파일을 더블클릭하면 브라우저에서 바로 열립니다.
이 상태로도 모든 기능이 동작해요. 단, 수정 내용은 **그 기기에만** 저장됩니다.
실시간 공유를 켜려면 아래 2~3단계를 진행하세요.

---

## 2단계 — 실시간 공유 켜기 (Firebase, 무료 · 약 5분)

1. https://console.firebase.google.com 접속 → 구글 로그인
2. **프로젝트 만들기** → 이름 아무거나(예: `crew-schedule`) → 애널리틱스는 꺼도 됨
3. 왼쪽 메뉴 **빌드 → Realtime Database** → **데이터베이스 만들기**
   - 위치: 아무거나(asia-southeast1 추천)
   - 보안 규칙: **테스트 모드로 시작** 선택 (친구끼리 쓰는 용도라 OK)
4. 왼쪽 위 ⚙️ **프로젝트 설정** → 아래로 스크롤 → **내 앱**에서 **웹(`</>`)** 아이콘 클릭
   - 앱 닉네임 입력 → 등록 → 화면에 뜨는 `firebaseConfig` 값 복사
5. `script.js`를 메모장으로 열어 상단의 이 부분을 복사한 값으로 교체:
   ```js
   const firebaseConfig = {
     apiKey:      "...",
     authDomain:  "....firebaseapp.com",
     databaseURL: "https://....firebasedatabase.app",   // 이 줄이 꼭 있어야 함!
     projectId:   "...",
   };
   const ROOM = "our-crew";   // 친구 그룹 이름. 모두 같은 값이어야 같은 스케줄을 봄
   ```
   > `databaseURL`이 복사한 값에 없으면 Realtime Database 페이지 상단에 있는
   > `https://...firebasedatabase.app` 주소를 직접 넣어주세요.
6. 저장 후 다시 열면 상단에 🟢 "실시간 공유 켜짐"이 뜨면 성공.

---

## 3단계 — 웹 링크로 배포해서 친구들에게 공유

서버 코드가 없는 정적 파일이라 **폴더를 드래그&드롭**만 하면 끝.

### 가장 쉬움 — Netlify Drop
1. https://app.netlify.com/drop 접속
2. `friends-schedule` 폴더를 통째로 드래그 → 끝
3. 생성된 주소(예: `https://random-name.netlify.app`)를 단톡방에 공유

### 대안 — Vercel
1. https://vercel.com 로그인 → **Add New → Project**
2. 이 폴더를 올리면 자동 배포, 주소 생성

> 친구들은 받은 링크만 누르면 폰/PC 브라우저에서 바로 사용. 설치·회원가입 없음.
> 한 명이 "스케줄 수정"에서 바꾸면 모두에게 즉시 반영됩니다.

## 사용법
- **📊 한눈에 보기** — 요일×사람 표. 초록=쉬는 날, 분홍=근무시간
- **🍻 약속 잡기** — 모일 친구 체크 → 요일별로 *다 같이 비는 시간* 표시. ⭐는 가장 여유로운 날
- **✏️ 스케줄 수정** — 시간 변경·휴무 토글·친구 추가/삭제 (자동 저장)

## 참고 (보안)
테스트 모드 규칙은 링크를 아는 사람이면 읽고 쓸 수 있어요. 친구끼리 쓰는 비공개 링크라 보통 충분하지만,
더 잠그고 싶으면 Firebase 콘솔의 Realtime Database **규칙**에서 인증을 추가하면 됩니다.
